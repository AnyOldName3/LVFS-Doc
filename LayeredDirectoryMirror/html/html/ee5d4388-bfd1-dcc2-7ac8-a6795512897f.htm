LVFSContextInfo Properties

The LVFSContextInfo type exposes the following members.

Properties
  NameDescription
Context
A dictionary holding arbitrary context data for each source. A source can store any or no data it wishes here, and it will be preserved for each file handle.
DeleteOnClose
Holds whether the file should be deleted when it is closed.
GetRequestor
Gets the Windows user requesting the operation. According to the DokanNet documentation, this method needs to be called in CreateFileHandle(String, FileAccess, FileShare, FileMode, FileOptions, FileAttributes, LVFSContextInfo)
IsDirectory
Holds whether the application software regards the file as a directory or regular file. If not done beforehand, this must be set in CreateFileHandle(String, FileAccess, FileShare, FileMode, FileOptions, FileAttributes, LVFSContextInfo)
NoCache
True if the file should not be cached, i.e. all reads come directly from the backing store, and all writes are immediately written through.
PagingIo
True if reads/writes are paging IO.
ProcessId
The process ID of the thread that originally requested the IO operation.
SynchronousIo
True if the operation is synchronous IO.
TryResetTimeout
Extends the timeout period for the current operation.
WriteToEndOfFile
True if the end of the file should be written to instead of the requested offset from the start.
